1. Executive Summary

Forge today is a strong single-developer, single-machine AI SDLC engine. The deep-dive of the codebase plus two strategic reference inputs — Google's skills marketplace pattern and the OpenHands "AI team of 131 specialists" model — surface five strategic deltas that block enterprise adoption and ecosystem reach.

3.2 Google skills model

  • Skills are markdown-only, self-contained, opt-in.
  • Installed via npx skills add <pack>.
  • Each skill is a directory with SKILL.md + assets.
  • Additive and reversible — never required.
Lesson: Forge's 21-agent catalog should follow the same shape — markdown packs, installable from a registry, never duplicated.

3.3 OpenHands & Noisy article

  • AGENTS.md = cross-runtime onboarding doc (Claude Code, Codex, Cursor, OpenHands, Gemini all read it).
  • Action-observation loop is the agent's contract, not a side effect.
  • Multi-agent delegation is multi-vendor (131-agent VoltAgent pack, Google skills).
  • MCP (14k+ servers) is the universal tool surface.
Lesson: Forge should emit AGENTS.md, expose its graph/verifier via MCP, log every action for replay, and consume external skill packs.

4. Non-Negotiable Compatibility Contract

Every phase must respect these. They are the reason this plan does not require a Forge 2.0 breaking migration.

  1. All 42 /forge-* slash commands keep their interface.
  2. Existing .forge/ paths keep their meaning. New artifacts live in new subdirs (audit/, actions/, secrets/, attestations/).
  3. Ledger Markdown format and proper-lockfile semantics preserved. New audit.jsonl is additive.
  4. Plan frontmatter schema preserved. New fields (capabilities, secrets_scope, actions_log) are optional.
  5. Code-graph schema migrations are non-destructive; v1 graphs remain readable.
  6. All new behavior gated by .forge/config.json keys with safe defaults.
  7. No Node major bump beyond current >=16.7.0. Optional deps lazy-loaded.
  8. CLAUDE.md agent directives (SENIOR DEV OVERRIDE, FORCED VERIFICATION, EDIT INTEGRITY) preserved.
P1 Shell-string elimination & input validation Effort: S Security

Problem

Multiple sites construct shell commands by template-string interpolation and pass them to execSync. CLI arguments and AI-generated task IDs flow into these unsanitized. The --since argument of the graph updater is a directly exploitable injection point.

Code change — replace template-string shell with array-form exec

Before · forge-graph/updater.js:139
const out = execSync(
  `git diff --name-status ${sinceRef}..HEAD`,
  { cwd: repoRoot, encoding: 'utf8' }
);
After · uses forge-cli/lib/exec.js
const { gitArgs } = require('../forge-cli/lib/exec');
const out = gitArgs(
  ['diff', '--name-status', `${validateRef(sinceRef)}..HEAD`],
  { cwd: repoRoot }
);
// validateRef rejects anything outside [A-Za-z0-9._/\-]{1,200}

What is improved

  • Eliminates shell injection across forge-graph/updater.js, forge-containers/orchestrator.js, worktree-orchestrator.js, forge-agents/provider.js.
  • Adds a single audit pattern: scripts/audit-shell-strings.sh greps for the anti-pattern in CI; any reintroduction fails the build.
  • CLI args validated with a 30-line argv parser that rejects unknown flags.

Benefits

  • CRITICAL fix Closes 4 of the 5 top-ranked exploitability paths from the deep-dive.
  • Zero observable behavior change for legitimate inputs.
  • No config flag — just-do-it fix.
  • CI gate prevents regression.

Files touched

  • forge-cli/lib/exec.js (new): gitArgs, dockerArgs, execFileWith wrappers.
  • forge-graph/updater.js, forge-containers/orchestrator.js, forge-containers/worktree-orchestrator.js, forge-agents/provider.js.
  • scripts/audit-shell-strings.sh (new): CI gate.

Risks & mitigations

Regression: an internal caller relied on shell features (globs, pipes)
Audit script flags every replacement during refactor. Any legitimate shell-feature use is moved into a named helper with explicit argv assembly. No silent globbing.
P2 Redaction + signed audit log + project identity Effort: M Security Enterprise

Problem

  • Ledger is Markdown — not queryable, not tamper-evident. SOC2 CC7 / ISO27001 A.12.4 require a structured audit trail.
  • .forge/knowledge/learnings.json accumulates forever and is re-injected into every future agent. Any credential ever surfaced lingers permanently.
  • No "actor" identity exists, so audit lines have no subject.

Code change — append-only signed audit record

Before · only ledger.md
// forge-session/ledger.js
appendDecision(`Phase 72 started by Claude at ${ts}`);
// no structured record, no signature, no chain
After · audit.js + identity.js
// forge-session/audit.js
const rec = {
  ts, actor: identity.subject(),
  action: 'phase.execute',
  subject: 'phase/72/PLAN.md',
  prev_hash, hash: sha256(prev_hash + canon(rec)),
};
rec.sig = ed25519.sign(rec.hash, identity.privateKey());
appendJsonl('.forge/audit/audit.jsonl', rec);

Code change — sensitive-data redactor

Before · raw write
fs.appendFileSync(LEDGER, `\n${entry}\n`);
fs.writeFileSync(LEARNINGS, JSON.stringify(state));
After · redact pass
const { redact } = require('./redactor');
// patterns: AWS keys, JWTs, sk-*, PEM, DB DSNs (lifted from forge-guard.js)
fs.appendFileSync(LEDGER, `\n${redact(entry)}\n`);
fs.writeFileSync(LEARNINGS, JSON.stringify(redactDeep(state)));
// secrets become «REDACTED:sha256[0..8]»

What is improved

  • Tamper-evident audit trail. Hash chain + Ed25519 signatures; forge-tools.cjs audit verify detects edits.
  • Project identity generated on /forge-init (Ed25519 keypair under .forge/identity/, 0600 mode). Optional identity-adapters/oidc.js accepts CI-injected OIDC ID token (FORGE_OIDC_TOKEN).
  • Secret leakage stopped at the file boundary — ledger and knowledge writes filter known credential shapes.
  • forge-tools.cjs knowledge scrub admin command retroactively redacts old data.

Benefits

  • Unblocks SOC2/ISO27001 audits Append-only + signed log satisfies the structured-audit control.
  • Compliance-grade identity Cleanly slots into enterprise SSO via OIDC adapter — no auth server required.
  • Redactor is safe-by-default and additive (no flag to enable).

Files touched

  • New: forge-session/{redactor,audit,identity}.js, forge-session/identity-adapters/{local,oidc,saml}.js.
  • Updated: forge-session/ledger.js, forge-session/knowledge.js, forge-cli/bin/forge-tools.cjs (new audit + knowledge scrub subcommands).
  • Refactor: hooks/dist/forge-guard.js imports shared patterns from redactor.js.

Risks & mitigations

Hash chain corruption from concurrent writes
Reuse existing proper-lockfile wrapper around audit.jsonl — same atomicity model as ledger.md.
Identity key loss locks user out of verifying their own history
identity.pub is committed; signatures verify against public key. Private key loss only prevents future signing. forge-tools.cjs identity rotate handles key rotation with a chained transition record.
Redactor false-positive removes legitimate content
Replacement is «REDACTED:<sha256-prefix>» — collisions detectable. Users can disable specific patterns in .forge/policy/redaction.yaml.

Code change — Docker hardening flags

Before · container-spec.js:163
function toDockerArgs(spec) {
  return [
    'run', '--rm',
    '--memory', spec.memory,
    '--cpus', spec.cpus,
    '--user', `${uid}:${gid}`,
    '-v', `${spec.workdir}:/work`,
    spec.image,
  ];
}
After · profiled hardening
function toDockerArgs(spec, profile = 'minimal') {
  const base = [
    'run', '--rm',
    '--memory', spec.memory,
    '--cpus', spec.cpus,
    '--user', `${uid}:${gid}`,
    '--cap-drop', 'ALL',
    '--security-opt', 'no-new-privileges',
    '--security-opt',
      `seccomp=${SECCOMP[profile]}`,
    '--pids-limit', '512',
    '--read-only',
    '--tmpfs', '/tmp:rw,size=512m',
    '--network', NET[profile],   // none|forge-egress
    '-v', `${spec.workdir}:/work`,
    spec.image,
  ];
  if (profile === 'build')
    base.push('--cap-add','CHOWN','--cap-add','DAC_OVERRIDE');
  return base;
}

Code change — allowlisted egress proxy

# forge-containers/egress-proxy/profiles/strict.yaml
allow:
  - api.anthropic.com
  - api.openai.com
  - registry.npmjs.org
  - "*.github.com"
  - objects.githubusercontent.com
default: deny
# Proxy binds to forge-egress bridge; containers use this network.

What is improved

  • Capability drop: agents lose NET_ADMIN, SYS_ADMIN, SETUID, etc.
  • No-new-privileges + seccomp: prevents kernel exploit escalation.
  • Read-only root fs + tmpfs: agent can only write to mounted workdir.
  • Egress allowlist: exfiltration to attacker-controlled hosts becomes impossible.
  • Env scoping: AWS/GCP/SSH agent socket no longer flow into agent subprocesses unless explicitly declared in plan frontmatter.

Benefits

  • Defense in depth Each layer (env, caps, network, fs) closes one exfil vector. Compromise requires breaking all four.
  • Procurement-ready Satisfies SOC2 CC6.6/CC6.7 and NIST 800-53 SC-7 controls.
  • Off by default Existing users see no change until containers.hardened: true is set.

Config flags

  • security.env_allowlist.enabled (default false in v1, true in v3)
  • containers.hardened (default false in v1, true in v2)
  • containers.egress.mode: off | allowlist | passthrough
  • containers.profile: minimal | build | research

Risks & mitigations

Egress allowlist breaks research agents that need npm install <new-pkg>
Three profiles: strict (only model + git + npm registry), build (adds CI/CD endpoints), research (broader allowlist + audit log). Each plan declares which it needs in frontmatter.
Seccomp profile breaks unusual workloads (e.g., node-gyp builds)
build profile uses a permissive seccomp. forge-doctor includes a "container syscall test" step that detects mismatch and recommends a profile bump.
Env allowlist breaks plans that legitimately need a secret
Plan declares secrets_scope: [ANTHROPIC_API_KEY] in frontmatter. warn mode logs unfulfilled needs without blocking; enforce mode rejects.

P5 Performance — tokenizers, async I/O, SQLite tuning Effort: S Performance

Problem

  • assessor.js:12 uses CHARS_PER_TOKEN = 4 as a universal heuristic. For code-heavy files true ratio is 3.2–3.8; for minified data, 2–3. This causes either unnecessary plan splits or context overflows.
  • fs.statSync in a loop per context file (assessor.js:81-83) and per-unit read-modify-write of metrics.json (metrics.js:13-20) bottleneck large repos.
  • forge-graph/query.js:112 enables WAL but never tunes cache_size or mmap_size.

Code change — pluggable tokenizers

Before · assessor.js:12
const CHARS_PER_TOKEN = 4;
function estimate(text) {
  return Math.ceil(text.length / CHARS_PER_TOKEN);
}
After · adapter selection
const { pickTokenizer } = require('./tokenizers');
const tokenizer = pickTokenizer(config.assess.tokenizer);
// 'heuristic' | 'anthropic' | 'tiktoken' | 'auto'
function estimate(text) {
  return tokenizer.count(text);
}
// anthropic/tiktoken loaded lazily; fallback to heuristic.

Code change — async assessor walk

Before
for (const f of files) {
  const st = fs.statSync(f);     // blocking
  total += Math.ceil(st.size/4);
}
After
const sizes = await Promise.all(
  files.map(f => fsp.stat(f).then(s => s.size))
);
total = sizes.reduce(
  (a,b) => a + tokenizer.estimateFromBytes(b), 0
);

Code change — SQLite tuning

// forge-graph/query.js — on db open
db.pragma('journal_mode = WAL');
db.pragma('cache_size = -64000');   // 64 MB
db.pragma('mmap_size  = 268435456');// 256 MB
db.pragma('temp_store = MEMORY');
db.pragma('synchronous = NORMAL');

What is improved

  • Accurate token estimation reduces unnecessary plan splits (faster end-to-end) and prevents context overflows (more reliable).
  • 10–100× faster file walks on large repos via concurrent stat.
  • Metrics batching (250 ms flush) eliminates the read-modify-write storm during 10-agent waves.
  • SQLite cache hot-path queries (graph impact/show) return in microseconds on warm cache.
  • Verification cache keyed by (file_hash, layer, config_hash) — re-runs of /forge-verify-work become near-instant on unchanged files.

Benefits

  • No format changes All artifacts identical on disk; only faster.
  • Lower variance in wave duration — less time spent on I/O blocking.
  • Opt-in tokenizer heuristic stays default; users opt-in to anthropic or tiktoken.

Risks & mitigations

tokenizer optional dep missing on air-gapped machine
Lazy import wrapped in try/catch — falls back to heuristic with a one-line warning.
Metrics batcher loses data on crash
Flush on SIGINT/SIGTERM + process.on('beforeExit'); flush interval 250 ms means worst-case loss is ~quarter-second of metrics, never decisions.

function span(name, attrs, fn) {
  if (!api) return fn();

Risks & mitigations

External skill packs ship malicious instructions
Skill-wrapper applies P4 capability model + P3 egress profile. skills add prints source URL + sha256 + signature status; user confirms before activation.
Runtime adapters drift in behavior, breaking replay
Adapter conformance test suite — every adapter runs the same 30-step fixture and produces normalized action records. CI gates new adapters.
MCP --write mode exposes destructive operations
Off by default. When on, every tool call requires a capability token (signed by project identity from P2) and writes to audit log.

11. Final Outcome — What Forge Becomes

Before (today, v1.20.4)

  • Strong single-machine SDLC engine.
  • Claude-Code-coupled execution.
  • Sensitive env + permissions bypass for every agent.
  • No identity, no audit, no SBOM, no signed releases.
  • 21 hardcoded personas; no marketplace.
  • < 10 % test coverage; no type-check; no observability export.

After (v2.0, all 8 phases shipped)

  • Hardened sandbox — scoped env, capability-gated agents, locked-down containers, egress-proxied.
  • Compliance-ready — signed append-only audit log, project identity, optional OIDC SSO, encryption-at-rest, SBOM + cosign.
  • Measurable — OTEL traces, action-observation logs, structured JSON logs, ≥ 60 % test coverage on core.
  • Multi-runtime — Claude Code, Codex, OpenHands, Gemini CLI — consuming external skill packs (Google, VoltAgent) while keeping the Forge SDLC verification wrapped around every one.
  • First-class MCP citizen — Forge's graph, verifier, impact analyzer callable from any external agent.
  • Every existing /forge-* command keeps working exactly as it does today.
Forge stops competing with Claude Code, Codex, or OpenHands and starts composing with all of them. The SDLC loop becomes Forge's product; the runtime becomes a configurable detail.
